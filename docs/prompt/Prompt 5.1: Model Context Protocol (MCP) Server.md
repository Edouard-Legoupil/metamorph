Implement an MCP (Model Context Protocol) server and API to provide agentic access to all curated knowledge, entities, wiki pages, cards, conflicts, and supporting evidence. The server must allow secure agent/bot access (API key, SSE support, rate limiting), expose core tools/functions for retrieving entities, searching over verified knowledge, getting conflicts, wiki or card content, and rendering context packs/resources as specified. Every response must include up-to-date provenance, confidence, and validation status, and the contract should match what the human UI would show. Provide resource endpoints for fetching context and evidence by entity, card, or outcome. Enable streaming updates, audit logging, webhooks for integration, and provide an SDK for agent consumers.

Verification & Test Guidance
- [ ] Verify mcp/server/service implements authentication (API key), SSE, rate limiting, and all required endpoints: get_entity, search_knowledge, get_conflicts, get_document_triplets, get_wiki_page, get_knowledge_card, and all resource fetches.
- [ ] Confirm agent/bot requests return accurate verified content, blocks/cards, and provenance/validation fields per contract, and that streaming/event endpoints work.
- [ ] Check presence of agent SDK, webhooks, and that all query types accepted in UI are also usable via API/MCP.
- [ ] Audit logs/records include all agent access and queries.
- [ ] PIPELINE.md and API.md document agentic/MCP functionality and expected response contracts.
